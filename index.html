Vercel & Netlify: Was sollte passieren?

Vercel

Vercel kann statische Projekte direkt aus dem Repo ausliefern. Bei Static‑Sites ist es meist „Import Repo → Deploy“.

Vercel: Neues Projekt → Repo auswählen → Framework: Other → Deploy

Wenn Vercel meckert, liegt’s fast immer an einer unerwarteten Ordnerstruktur oder einer falschen Publish‑Directory.

Netlify

Netlify ist bei Static‑Sites ebenfalls entspannt. netlify.toml sagt: kein Build, Publish ist Root.

[build]
  publish = "."
  command = ""

Wenn du später ein Framework nutzt, packst du hier Build‑Command und Publish‑Ordner rein.

Mini‑FAQ

Warum kein Next.js / Vite / NPM?

Weil du zuerst die Kette ZIP → Repo → Hosting testen willst. Builds sind der Ort, wo Chaos wohnt.

Kann ich daraus später ein echtes Projekt machen?

Klar. Ersetze die Static‑Files durch dein Framework‑Projekt – und passe Netlify/Vercel‑Config an.

Warum gibt es vercel.json und netlify.toml überhaupt?

Sie sind optionale „Hints“, damit die Plattformen nicht kreativ werden. Kreativität ist beim Deploy selten hilfreich.
